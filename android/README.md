# ONF Android — Flagship Offline Alpha

This directory contains the standalone native Android edition of Offline Neural Facilitator. It is designed for flagship foldables and phones, with the Samsung Galaxy Z Fold7 as the primary physical test target.

## What is implemented

- Native Kotlin and Jetpack Compose workspace; no WebView and no Python/FastAPI process.
- Adaptive two-pane unfolded layout and compact cover-display navigation.
- Canonical sessions, transcripts, guidance, decisions, actions, risks, titles, and metrics.
- Deterministic decision/action/risk capture before probabilistic generation.
- SQLite session snapshots and a section-aware local knowledge vault.
- Five bundled Markdown facilitator skills and the fictional Northstar Code Blue evidence pack.
- Foreground `AudioRecord` capture in five-second PCM/WAV segments.
- Independent AES-256-GCM envelopes with per-segment IVs generated by Android Keystore.
- Atomic encrypted audio spool in app-private storage.
- Detection of the official Microsoft Foundry Local Android companion preview.
- A retained private model library for importing, selecting, switching, and explicitly removing multiple Gemma 4 `.litertlm` packs.
- Detection of Google AI Edge Gallery with an explicit explanation of Android's cross-app model boundary.
- LiteRT-LM 0.14.0 integration for imported Gemma 4 models.
- Bounded Gemma generation with GPU-first loading, CPU fallback, and optional multimodal audio transcription.
- Local JSON and Markdown exports.
- No `INTERNET` permission, analytics, telemetry, cloud backup, or device-transfer backup.

## Install or build a sideload APK

### Recommended tester installation order

1. Install the official [Google AI Edge Gallery](https://play.google.com/store/apps/details?id=com.google.ai.edge.gallery) from Google Play first. Use it as the alpha programme's model-discovery and device-validation companion to explore Gemma variants and confirm that local AI is viable on the phone.
2. Download and verify the ONF APK below, then sideload it. Grant **Install unknown apps** only to the browser or file manager opening the APK.
3. Open **ONF → Menu → Private system and models**. Import an original/shared LiteRT-LM-compatible `.litertlm` model pack with Android's document picker if generative reasoning or local model transcription is required.
4. Edge Gallery's own downloaded weights remain isolated in Edge Gallery's app-specific storage. ONF cannot silently reuse them, and Gallery currently offers no supported cross-app serving/export API. Installing Gallery first does **not** remove ONF's explicit model-import step.
5. Optionally install the official [Microsoft Foundry Local Android companion preview](https://play.google.com/store/apps/details?id=com.microsoft.foundrylocal.app). ONF detects it, but model-catalog and inference integration remain disabled until Microsoft provides the documented, gated Android client SDK.

In short: Edge Gallery is the recommended tester reference companion today; Foundry Local is the intended future service companion. ONF's deterministic workspace, bundled knowledge, skills, decisions, actions, and exports remain usable without either generative runtime.

## Connect and install on the Fold7

1. On the phone, open **Settings → About phone → Software information**.
2. Tap **Build number** seven times.
3. Open **Settings → Developer options** and enable **USB debugging**.
4. Connect the unlocked phone by USB and accept its RSA authorization prompt.
5. Verify that Windows sees it:

```powershell
& "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe" devices -l
```

6. Build and install:

```powershell
.\android\scripts\build.ps1 -FullValidation
.\android\scripts\install.ps1
```

The install script prefers a physical device over an emulator. Use `-Serial` if more than one physical device is attached.

## Foundry Local Android preview

Microsoft announced Foundry Local for Android as a gated preview in November 2025. Its Play Store package, `com.microsoft.foundrylocal.app`, is a companion service that owns model catalog downloads, model management, and local inference for client apps using the official Foundry Local Android SDK.

The current companion exposes an exported Android service and advertises idiomatic Kotlin chat and audio APIs. The client SDK is not published on Maven Central or Google Maven; Microsoft directs developers to the gated preview enrollment at <https://aka.ms/foundrylocal-androidprp>. ONF therefore does not call undocumented Binder interfaces or redistribute code from the companion APK.

ONF's intended runtime order is:

1. **Foundry Local Android SDK**, when Microsoft grants the gated client package;
2. **LiteRT-LM**, the currently integrated and physically verified fallback;
3. deterministic session, RAG, skills, decisions, and actions when no model runtime is available.

The System sheet detects and displays the installed companion version, opens the official companion, and links to Microsoft's SDK enrollment. It does not present Foundry catalog models as usable until the documented client SDK is linked. ONF keeps its own manifest free of the `INTERNET` permission; a future official IPC integration can let the companion own catalog traffic while meeting inference remains on-device.

The preview companion has its own `INTERNET` permission and license/data terms. Its Play disclosure says it may collect app activity, app information/performance, and device identifiers, although model inference is described as on-device and no data is shared with third parties. Any production ONF integration must complete a privacy assessment and expose the companion/telemetry state before claiming the same zero-telemetry boundary as the current LiteRT path.

See [FOUNDRY_LOCAL_ANDROID.md](FOUNDRY_LOCAL_ANDROID.md) for verified package details, the adapter plan, privacy gates, and acceptance criteria.

## Import Gemma 4

The APK does not bundle multi-gigabyte model weights.

1. Copy a LiteRT-LM-compatible `.litertlm` model to the phone.
2. Open **System** (or **Menu → Private system and models** on the cover display).
3. Select **Import Gemma model** or **Import another model**.
4. Choose the model file.
5. Leave the app open while it copies the model into private app storage and initializes GPU inference.

Imports are retained. Loading E2B no longer deletes E4B, and loading E4B no longer deletes E2B. The Private System sheet shows every ONF-owned model, its size, active state, and explicit **Load**/**Remove** actions. A failed switch attempts to restore the previously selected model.

Recommended starting point for the Fold7:

- Gemma 4 E2B: approximately 2.58 GB and the first performance target.
- Gemma 4 E4B: approximately 3.65 GB and the higher-quality optional target.

The app attempts GPU initialization first and falls back to CPU. A multimodal Gemma variant can transcribe the newest encrypted WAV segment through LiteRT-LM.

### Models already downloaded by another app

ONF detects Google AI Edge Gallery, but it cannot silently open Gallery's downloaded model files. Gallery stores models in its own app-specific `Android/data` directory; Android 11 and later block both another application and the system document picker from browsing that directory. Gallery also exposes no supported cross-app inference service or model-export contract.

Production choices are therefore explicit:

1. import the original/shared `.litertlm` file through Android's document picker; or
2. use Microsoft Foundry Local through its official companion SDK once Microsoft grants the gated client artifact.

For this development device only, ADB was used to copy Gallery's E4B file into ONF's private sandbox. That is a test provisioning technique, not an application feature or a production data-access path.

## Privacy design

- No network permission is declared.
- Audio, sessions, knowledge, and imported models live under the Android application sandbox.
- Audio is encrypted before each segment is finalized.
- The AES key is non-exportable and generated by Android Keystore.
- Android cloud backup and device-transfer backup are disabled.
- Explicit exports use the system document picker so the user chooses the destination.
- Uninstalling the app normally removes the private model, session, knowledge, and encrypted audio stores.

## Validation

Local and device checks:

```powershell
Push-Location android
.\gradlew.bat :app:testDebugUnitTest
.\gradlew.bat :app:lintDebug
.\gradlew.bat :app:connectedDebugAndroidTest
Pop-Location
```

Run `connectedDebugAndroidTest` on an emulator or an unprovisioned test device. The Android Gradle managed test flow may uninstall the target package during cleanup, which deletes app-private sessions, keys, audio, knowledge, and multi-gigabyte models. On a provisioned Fold7, install the app/test APKs with `adb install -r -t` and invoke the selected instrumentation classes directly so private data is preserved.

The test suite covers deterministic facilitation, titles, actions, decisions, local RAG ranking, Markdown section citations, session serialization, WAV encoding, AES-GCM tamper detection, Android Keystore round trips, real foreground microphone capture, launch behavior, the Code Blue showcase, retained-model selection/removal, runtime-chooser rendering, companion/Gallery detection, and provisioned physical E2B/E4B switch probes.

The APK is also checked for:

- Android v2 signature validity;
- no `INTERNET` permission;
- arm64-v8a and x86_64 native libraries;
- 16 KB ZIP alignment;
- 16 KB ELF LOAD alignment for every arm64 library.
